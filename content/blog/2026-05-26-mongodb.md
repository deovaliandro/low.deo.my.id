## Soal 1: CREATE

### 1. Tambahkan 2 penulis baru ke koleksi `penulis`

Untuk menambahkan 2 penulis baru maka kita menggunakan query `insertMany`. Untuk
isinya berupa `_id` diambil 5 dan 6 karena 1-4 sudah ada dan untuk menghindari
error karena id yang tidak unik. Untuk `nama`, `spesialisasi`, `tahun_aktif` dan
`negara` menggunakan data dari Wikipedia.

```js
db.penulis.insertMany([
  {
    _id: 5,
    nama: "Fiersa Besari",
    spesialisasi: ["Fiksi", "Romansa", "Petualangan"],
    tahun_aktif: 2016,
    negara: "Indonesia"
  },
  {
    _id: 6,
    nama: "Raditya Dika",
    spesialisasi: ["Komedi", "Romansa"],
    tahun_aktif: 2005,
    negara: "Indonesia"
  }
]);
```

Hasilnya terdapat 2 penulis baru di koleksi penulis (`_id` 5 dan 6):

![1](/images/20251129085507.png)

### 2. Tambahkan 3 buku baru dari penulis yang berbeda ke koleksi `buku`

error karena id yang tidak unik. Untuk `judul`, `genre`, `harga`, `stok`,
`tahun_terbit` dan `rating` menggunakan data dari Wikipedia. Untuk `penulis_id`,

```js
    genre: "Romansa",

  {

    genre: "Romansa",
```

## Soal 2: READ

Untuk menampilkan semua buku dengan genre fiksi, kita menggunakan query `find`.
Untuk menampilkan hanya genre fiksi saja, maka diberikan query `genre: "Fiksi"`.

```js
db.buku.find({
  genre: "Fiksi"
});
```

Hasilnya adalah semua buku dengan genre fiksi:

### 2. Tampilkan buku dengan harga di atas Rp80.000

menggunakan query `find` seperti query sebelumnya, bedanya pada harga yang
diberikan menggunakan `$gt` (greater than) untuk menentukan harga yang lebih
besar dari Rp80.000.

```js
db.buku.find({
  harga: { $gt: 80000 }
});
```

dari 2010.

```js
});
```

## Soal 3: UPDATE

### 1. Update harga buku "Laskar Pelangi" menjadi Rp 80.000

Untuk mengupdate buku Laskar Pelangi kita menggunakan query `updateOne` karena
hanya satu item yang akan kita update. Hal pertama yang dilakukan adalah
menentukan buku yang mana yang akan kita update:

1. Kita bisa menggunakan `_id: 1` karena buku Laskar Pelangi memiliki `_id: 1`.
2. Kita bisa meggunakan `judul: "Laskar Pelangi"` untuk memudahkan jika kita
   tidak mengetahui `_id` dari buku tersebut.

Cara pertama:

```js
db.buku.updateOne(
  { _id: 1 },
  {
    $set: {
      harga: 80000
    }
  }
);
```

Cara kedua:

```js
db.buku.updateOne(
  { judul: "Laskar Pelangi" },
  {
    $set: {
      harga: 80000
    }
  }
);
```

Hasil dari cara pertama dan kedua akan sama. Hasilnya adalah buku Laskar Pelangi
berubah nilai harganya menjadi Rp80.000:

![8](/images/20251129091351.png)

### 2. Tambahkan stok sebanyak 10 unit untuk semua buku genre "Fantasi"

Untuk mengupdate stok sebanyak 10 unit untuk semua buku genre fantasi maka kita
menggunakan query `updateMany` karena itemnya banyak. Hal pertama yang kita atur
adalah menentukan item mana saja yang akan di update, dalam hal ini semua buku
dengan genre Fiksi. Kemudian setelah itemnya sudah terfilter, diberikan query
`$inc` (increment) yaitu perintah untuk menambah nilai. Nilai yang ditambah
dalam hal ini adalah stok.

```js
db.buku.updateMany(
  { genre: "Fiksi" },
  {
    $inc: {
      stok: 10
    }
  }
);
```

Hasilnya adalah semua buku dengan genre fiksi bertambah:

![9](/images/20251129092035.png)

### 3. Update status transaksi dengan ID 7 menjadi "selesai"

Untuk mengupdate transaksi id 7 kita menggunakan query `updateOne` karena hanya
satu item yang akan kita update. Hal pertama yang dilakukan adalah menentukan
transaksi dengan `_id: 7`, lalu kita melakukan `$set` untuk `status` menjadi
`selesai`.

```js
db.transaksi.updateOne(
  { _id: 7 },
  {
    $set: {
      status: "selesai"
    }
  }
);
```

Hasilnya adalah transaksi dengan `_id: 7` berubah status menjadi selesai:

![10](/images/20251129112503.png)

## Soal 5: Aggregation Pipeline - Analisis Buku

### 1. Menghitung jumlah buku per genre

Untuk menghitung jumlah buku per-genre maka kita menggunakan query `aggregate`
dan memanfaatkan query `$group`. Group akan membuat group-group tergantung dari
id yang diberikan. Dalam hal ini group menggunakan kategori genre. Dan untuk
setiap item yang digabungkan ke group tersebut maka ditambahkan 1 pada `count`
dengan menggunakan query `$sum`.

Setelah pengroupan dilakukan, maka kita mengurutkannya dengan query `$sort`
dengan nilai 1, berarti kita berdasarkan urutan nilai terkecil ke terbesar.

```js
db.buku.aggregate([
  {
    $group: {
      _id: "$genre",
      count: { $sum: 1 }
    }
  },
  {
    $sort: { count: 1 }
  }
]);
```

Hasilnya adalah urutan genre dengan total buku per genrenya:

![13](/images/20251129115346.png)

### 2. Menghitung rata-rata harga buku per genre

Untuk menghitung rata-rata harga buku per-genre maka kita menggunakan query
`aggregate` dan memanfaatkan query `$group`. Group akan membuat group-group
tergantung dari id yang diberikan. Dalam hal ini group menggunakan kategori
genre. Dan untuk setiap item genre akan dilakukan perhitungan rata-rata harga
yang menggunakan perintah `$avg`. Data rata-rata ini disimpan di dalam key
`avgPrice`.

Setelah pengroupan dilakukan, maka kita mengurutkannya dengan query `$sort`
dengan nilai 1, berarti kita berdasarkan urutan nilai terkecil ke terbesar.

```js
db.buku.aggregate([
  {
    $group: {
      _id: "$genre",
      avgPrice: { $avg: "$harga" }
    }
  },
  {
    $sort: { avgPrice: 1 }
  }
]);
```

Hasilnya adalah group genre yang berisi nilai rata-rata setiap bukunya:

![14](/images/20251129120451.png)

### 3. Menemukan buku dengan harga tertinggi dan terendah

Untuk menemukan buku dengan harga tertinggi, kita bisa menggunakan dua cara:

Cara pertama dengan menggunakan `group` tetapi kelemahannya hanya akan
menampilkan harga tertinggi dan harga terendah tetapi tidak menampilkan judul
bukunya. Kita menggunakan query `$max` dan `$min`.

```js
db.buku.aggregate([
  {
    $group: {
      _id: null,
      max: { $max: "$harga" },
      min: { $min: "$harga" }
    }
  }
]);
```

Hasilnya adalah nilai buku yang memiliki nilai tertinggi (max) dan nilai
terendah (min):

![15](/images/20251129120644.png)

Cara kedua dengan menggunakan facet[^1]. `facet` adalah query multi agregasi dalam satu agregasi. Kita menggunakan agregasi max, dimana langkah pertama adalah melakukan sorting berdasarkan harga dari tertinggi ke terendah, kemudian diambil item pertama dengan memanfaatkan `limit: 1`. Lalu dibuat data dengan mengambil judul dan harganya, `_id: 0` berarti kita tidak mengambil id-nya. Begitu pula dengan min, perbedaannya ada di pengurutannya yang mengambil dari harga terendah ke harga terbesar.

```js
db.buku.aggregate([
  {
    $facet: {
      max: [
        { $sort: { harga: -1 } },
        { $limit: 1 },
        { $project: { _id: 0, judul: "$judul", harga: "$harga" } }
      ],
      min: [
        { $sort: { harga: 1 } },
        { $limit: 1 },
        { $project: { _id: 0, judul: "$judul", harga: "$harga" } }
      ]
    }
  }
]);
```

Hasilnya adalah buku dengan nilai tertinggi dan terendah, lengkap dengan judul dan harganya:

![16](/images/20251129121057.png)

[^1]: <https://www.mongodb.com/docs/manual/reference/operator/aggregation/facet/>

## Soal 6: Aggregation Pipeline - Analisis Penjualan

### 1. Total pendapatan dari semua transaksi yang selesai

Untuk menghitung total total pendapatan dari semua transaksi yang selesai maka
kita gunakan `match` untuk memfilter semua yang berstatus selesai. Kemudian
dilakukan group yang menggabungkan item yang telah difilter sebelumnya. Pada
query diberikan`_id: null` berarti kita mengambil semua data. Lalu dilakukan
`$sum` pada semua `total_harga`. Nilainya disimpan di dalam key `total`.

```js
db.transaksi.aggregate([
  {
    $match: { status: "selesai" }
  },
  {
    $group: {
      _id: null,
      total: { $sum: "$total_harga" }
    }
  }
]);
```

Hasilnya adalah total nilai dari transaksi yang selesai:

![17](/images/20251129125945.png)

### 2. Jumlah transaksi per buku (join dengan koleksi buku)

Untuk membuat jumlah transaksi per buku maka pertama yang kita lakukan adalah
join antara koleksi transaksi dengan koleksi buku dengan menggunakan perintah
`lookup` dimana yang menjadi penanda di koleksi transaksi adalah `buku_id` dan
menjadi penanda di koleksi buku adalah `_id`.

Setelah dilakukan join, maka dilakukan `unwind` untuk memecah elemen array
menjadi lebih sederhana.

Setelah itu kemudian dilakukan `group` berdasarkan judul buku dan ditambahkan
key `totalTerjual` untuk menampung jumlah transaksi setiap buku.

```js
db.transaksi.aggregate([
  {
    $lookup: {
      from: "buku",
      localField: "buku_id",
      foreignField: "_id",
      as: "buku"
    }
  },
  {
    $unwind: "$buku"
  },
  {
    $group: {
      _id: "$buku.judul",
      totalTerjual: { $sum: "$jumlah" },
    }
  }
]);
```

Hasilnya adalah nama setiap buku dan jumlah transaksi per buku:

![18](/images/20251129143938.png)

### 3. Buku terlaris (berdasarkan jumlah terjual)

Untuk menentukan buku terlaris, maka langkahnya seperti sebelumnya tetapi
kemudian ditambahkan sorting dari nilai terbesar ke nilai terkecil dengan
menggunakan fungsi `sort` pada `totalTerjual` dengan menggunakan `-1`. Langkah
terakhir dilakukan dengan mengambil item pertama dengan menggunakan fungsi
`limit`.

```js
db.transaksi.aggregate([
  {
    $match: {
      status: "selesai"
    }
  },
  {
    $lookup: {
      from: "buku",
      localField: "buku_id",
      foreignField: "_id",
      as: "buku"
    }
  },
  {
    $unwind: "$buku"
  },
  {
    $group: {
      _id: "$buku.judul",
      totalTerjual: { $sum: "$jumlah" },
    }
  },
  {
    $sort: { totalTerjual: -1 }
  },
  {
    $limit: 1
  }
]);
```

Hasilnya adalah buku dengan penjualan terbanyak:

![19](/images/20251129145105.png)

## Soal 7: Aggregation Pipeline - Analisis Lanjutan

### 1. Rata-rata rating buku per penulis (join dengan koleksi penulis)

Untuk menghitung rata-rata rating buku per-penulis maka kita perlu join antara
koleksi buku dengan koleksi penulis. Dengan memanfaatkan query `lookup` yang
menggunakan penanda `penulis_id` pada koleksi buku dan `_id` pada koleksi
penulis.

Setelah dilakukan join, maka langkah berikutnya adalah dilakukan `unwind` untuk
memecah array menjadi lebih sederhana.

Kemudian langkah berikutnya dilakukan group berdasarkan nama penulis dan dibuat
satu key baru yaitu `avgRating` yang menampung nilai rata-rata rating dengan
memanfaatkan query `avg`.

```js
db.buku.aggregate([
  {
    $lookup: {
      from: "penulis",
      localField: "penulis_id",
      foreignField: "_id",
      as: "buku"
    }
  },
  {
    $unwind: "$buku"
  },
  {
    $group: {
      _id: "$buku.nama",
      avgRating: { $avg: "$rating" }
    }
  }
]);
```

Hasilnya adalah daftar penulis beserta dengan rata-rata ratingnya:

![20](/images/20251129145151.png)

### 2. Total pendapatan per bulan

Untuk menghitung total pendapatan per bulan pertama-tama dilakukan filter khusus
unuk transaksi yang statusnya selesai dengan menggunakan query `match`. Langkah
berikutnya adalah kita melakukan group data berdasarkan bulan dan tahunnya,
kemudian dibuat satu key yang menampung total harga dengan nama
`totalPendapatan` dengan memanfaatkan query `sum`.

Langkah berikutnya kita melakukan sort dari tahun terkecil dan bulan terkecil
berurutan sampai di tahun terbesar dan bulan terbesar.

Langkah terakhir adalah menampilkna nilai tahun, bulan dan total pendapatannya.

```js
db.transaksi.aggregate([
  {
    $match: { status: "selesai" }
  },
  {
    $group: {
      _id: {
        tahun: { $year: "$tanggal" },
        bulan: { $month: "$tanggal" }
      },
      totalPendapatan: { $sum: "$total_harga" }
    }
  },
  {
    $sort: {
      "_id.tahun": 1,
      "_id.bulan": 1
    }
  },
  {
    $project: {
      _id: 0,
      Tahun: "$_id.tahun",
      Bulan: "$_id.bulan",
      TotalPendapatan: "$totalPendapatan"
    }
  }
]);
```

Hasilnya adalah daftar tahun beserta dengan bulannya dengan total pendapatan
pada bulan tersebut:

![21](/images/20251129145231.png)

### 3. Penulis dengan total penjualan tertinggi

Untuk menghitung penulis dengan total penjualan tertinggi maka perlu dilakukan 2
kali join.

Langkah pertama tetap yaitu memfilter transaksi yang berstatus selesai dengan
menggunakan query `match`.

Langkah berikutnya adalah join pertama antara transaksi dengan koleksi buku
dengan memanfaatkan penanda `buku_id` pada koleksi transaksi dan `_id` pada
koleksi buku. Setelah dilakukan join, maka dilakukan `unwind` untuk memecah
array.

Join yang kedua dilakukan antara hasil join transaksi dan buku dengan koleksi
penulis. Dengan memanfaatkan penanda `detailBuku.penulis_id` dengan `_id` di
koleksi penulis kita bisa melakukan join tersebut. Seperti pada join pertama,
setelah join dilakukan maka dilakukan `unwind` untuk memecah array.

Langkah berikutnya adalah kita melakukan group penulis dengan menggunakan query
`group` dengan berdasar pada nama penulis pada key `$detailPenulis.nama` dan
ditambahkan satu key baru yaitu `totalTerjual` yang berisi `sum` dari jumlah
penjualan.

Langkah berikutnya adalah kita melakukan sorting dari group sebelumnya dari item
yang memiliki `totalTerjual` terbesar ke terkecil.

Langkah terakhirnya adalah mengambil nilai pertama dengan memanfaatkan query
`limit: 1`.

```js
db.transaksi.aggregate([
  {
    $match: { status: "selesai" }
  },
  {
    $lookup: {
      from: "buku",
      localField: "buku_id",
      foreignField: "_id",
      as: "detailBuku"
    }
  },
  {
    $unwind: "$detailBuku"
  },
  {
    $lookup: {
      from: "penulis",
      localField: "detailBuku.penulis_id",
      foreignField: "_id",
      as: "detailPenulis"
    }
  },
  {
    $unwind: "$detailPenulis"
  },
  {
    $group: {
      _id: "$detailPenulis.nama",
      totalTerjual: { $sum: "$jumlah" }
    }
  },
  {
    $sort: { totalTerjual: -1 }
  },
  {
    $limit: 1
  }
]);
```

Hasilnya adalah nama penulis dengan penjualan tertinggi beserta dengan total
buku yang terjual:

![22](/images/20251129145326.png)
